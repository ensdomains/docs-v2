# CCIP

[EIP-3668](https://eips.ethereum.org/EIPS/eip-3668), otherwise known as CCIP Read, is an Ethereum standard that enables any smart contract to load data from an external source (such as a database, rollup, L2, or http).
This means that using CCIP we can load information about names and subnames, from for example an L2, Database, or a regular REST endpoint.

## So unlimited free subnames?

By loading data from a gateway, write operations (and thereby gas costs) can be deferred to an L2, or completely circumvented using a database.
This means subnames can be issued gaslessly, tokenized on an L2, inferred from live DNS data, and more.
